## Take your PostgreSQL-backed apps to the next level

- **Blog:** Blog
- **URL:** https://devblogs.microsoft.com/blog/take-your-postgresql-backed-apps-to-the-next-level

### 要約
PostgreSQL を Microsoft の開発者向け製品群と組み合わせて活用する方法を紹介する投稿です。Azure 上の PostgreSQL、Visual Studio Code の拡張、agentic development、AI サービスをつなげて使う流れが中心です。あわせて、PostgreSQL Like a Pro という動画シリーズを案内しています。シリーズでは、MCP を使ったエージェントのデータ探索、Oracle から Azure Database for PostgreSQL への AI 支援移行、Azure 上での性能最適化、Azure HorizonDB のスケーリング特性が扱われます。PostgreSQL を単独の DB としてではなく、開発から運用までの一連の体験として捉えている内容です。

### 開発者にとって重要なこと
PostgreSQL を使うアプリや移行プロジェクト、エージェント連携を進めるチームに関係します。VS Code での移行支援や、Azure での性能・可用性設計を見直すきっかけになります。特に、データアクセスを AI ワークフローに組み込む予定がある場合は、MCP と vector search の使い方が参考になります。新しい基盤や移行手順を検討しているチームほど影響が大きい投稿です。

### アクションプラン
- **⚠️** PostgreSQL を使うアプリの移行先、性能要件、運用要件を整理する。
- VS Code の移行支援や PostgreSQL 関連拡張を、既存の開発フローに組み込めるか確認する。
- エージェントや AI 連携を計画している場合は、MCP と vector search の利用可否を検討する。
- Azure Database for PostgreSQL や Azure HorizonDB の採用判断が必要なら、本文と動画シリーズを参照して比較する。

## New Aspire database integrations

- **Blog:** Aspire
- **URL:** https://devblogs.microsoft.com/aspire/aspire-new-database-integrations

### 要約
Aspire 13.2 の新しいデータベース統合を紹介する投稿です。MongoDB 向けに Entity Framework Core クライアント統合が追加され、C# で MongoDB を扱いやすくなりました。Azure Data Lake Storage の統合も追加され、分析や ETL 系のワークロードを Aspire でつなぎやすくなっています。加えて、MongoDB 接続文字列の修正、SQL Server の追加エクスポート、各種 emulator の更新が案内されています。Redis の既定スキームが rediss:// に変わり、暗号化が標準になった点も重要です。

### 開発者にとって重要なこと
Aspire を使っているアプリのデータ層やローカル開発環境に直接関係します。MongoDB を C#/EF Core で扱いたいチームや、Data Lake を含む分析基盤を組み込みたいチームには特に有用です。emulator 更新や Redis の既定変更は、ローカル実行や接続設定に影響する可能性があります。Aspire 13.2 への更新判断をする際の確認ポイントがまとまった投稿です。

### アクションプラン
- Aspire 13.2 への更新で、MongoDB EF Core 統合や ADLS 統合を採用できるか確認する。
- **⚠️** MongoDB 接続文字列や Redis 接続設定への影響をチェックし、既存の回避策が不要か見直す。
- SQL Server や emulator を使っている場合は、更新内容がローカル開発・CI に与える影響を確認する。
- データ分析や ETL を Aspire に寄せる予定があるなら、ADLS 統合の導入可否を本文と release notes で確認する。
