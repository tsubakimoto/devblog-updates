## SharePoint Framework (SPFx) roadmap update – April 2026

- **Blog:** Microsoft365dev
- **URL:** https://devblogs.microsoft.com/microsoft365dev/sharepoint-framework-spfx-roadmap-update-april-2026

### 要約
SharePoint Framework と Microsoft 365 拡張エコシステムの 4 月時点のロードマップ更新です。SPFx 1.23 は release candidate になっており、品質確認と npm 脆弱性への対応が進められています。次の 1.24 では、AI を活用した新機能が public preview に入る計画です。あわせて、今後は四半期ごとのリリースサイクルに移行し、予測しやすさを高める方針が示されています。コミュニティイベントや次の発表予定も含め、今後の SPFx の方向性をまとめた内容です。

### 開発者にとって重要なこと
SPFx を使って SharePoint や Microsoft 365 向けのソリューションを作っているチームに直接関係します。release candidate の検証や npm 脆弱性対応は、既存ソリューションの互換性確認に影響します。AI 関連の新機能やリリース周期の変更は、今後の実装計画や移行スケジュールにも関わります。

### アクションプラン
- **⚠️** SPFx 1.23 RC を使うソリューションを検証し、互換性や npm 脆弱性の影響を確認する。
- 1.24 の AI プレビュー計画を追跡し、将来の機能採用候補として評価する。
- 四半期リリースへの移行を踏まえ、社内の更新計画とテストタイミングを見直す。

### リファレンス
- [SharePoint Framework (SPFx) roadmap update – April 2026](https://devblogs.microsoft.com/microsoft365dev/sharepoint-framework-spfx-roadmap-update-april-2026)

## Welcome to SkiaSharp 4.0 Preview 1

- **Blog:** .NET
- **URL:** https://devblogs.microsoft.com/dotnet/welcome-to-skia-sharp-40-preview1

### 要約
SkiaSharp 4.0 Preview 1 の公開を告知する投稿です。長年使われてきた SkiaSharp に、Skia エンジンの大きな更新と新しい API が入ります。variable fonts、color font palettes、SKPathBuilder などの新機能が追加されています。Linux Bionic や Tizen 64-bit への対応も含まれ、プラットフォーム面でも拡張されています。Uno Platform が共同メンテナーに加わる点も大きな変化です。

### 開発者にとって重要なこと
.NET MAUI、WebAssembly、WinUI 3 などで SkiaSharp を使う開発者に関係します。レンダリング品質、性能、セキュリティに加えて、API 変更の影響確認が必要です。共同メンテナ体制の変更は、今後の保守やリリース速度にも影響します。

### アクションプラン
- **⚠️** 既存の描画コードを Preview 1 で試し、レイアウトやレンダリングの差分を確認する。
- variable fonts や新しいパス構築 API を使う予定があるなら、移行設計を先に検討する。
- .NET 依存アプリでは、更新計画とフィードバック提出先をチームで共有する。

### リファレンス
- [Welcome to SkiaSharp 4.0 Preview 1](https://devblogs.microsoft.com/dotnet/welcome-to-skia-sharp-40-preview1)

## High-Performance Distributed Caching with .NET and Postgres on Azure

- **Blog:** .NET
- **URL:** https://devblogs.microsoft.com/dotnet/high-performance-distributed-caching-dotnet-postgres-azure

### 要約
.NET 10 のコンソールアプリを題材に、Postgres を使った分散キャッシュと HybridCache を組み合わせる方法を解説する投稿です。Generic Host、設定、秘密情報の管理、ログ、バックグラウンドサービスを段階的に組み立てています。Azure Database for PostgreSQL を分散キャッシュとして使い、インメモリキャッシュと組み合わせる構成が中心です。キャッシュの有効期限やヒット時の計測も含まれており、性能改善の見え方が分かります。実運用へ持ち込みやすいパターンを示すデモです。

### 開発者にとって重要なこと
低レイテンシと耐障害性が必要な .NET アプリや API を作るチームに関係します。既に Postgres を使っているなら、分散キャッシュ基盤を追加しやすい内容です。秘密情報の扱いや設定分離の実践例としても参考になります。

### アクションプラン
- Postgres を既に使っているサービスで、HybridCache 導入の効果を評価する。
- **⚠️** 接続文字列や秘密情報を user-secrets / 環境変数 / Key Vault で管理する前提に整える。
- キャッシュ期限とヒット率を測定し、API やバックグラウンド処理の待ち時間改善を確認する。

### リファレンス
- [High-Performance Distributed Caching with .NET and Postgres on Azure](https://devblogs.microsoft.com/dotnet/high-performance-distributed-caching-dotnet-postgres-azure)

## PowerToys 0.99 is here: new monitor controls, easier window management, and Dock upgrades

- **Blog:** Commandline
- **URL:** https://devblogs.microsoft.com/commandline/powertoys-0-99-is-here-new-monitor-controls-easier-window-management-and-dock-upgrades

### 要約
PowerToys 0.99 の新機能と改善点をまとめたリリース告知です。Grab And Move でウィンドウ移動・リサイズがしやすくなり、Power Display でモニター設定をトレイから操作できます。Command Palette と Dock には、履歴、固定、コンパクト表示、安定性改善などの更新が入っています。Keyboard Manager や ZoomIt、Image Resizer、Advanced Paste などにも実用的な改良があります。PowerToys を日常的に使う Windows 開発者向けの更新です。

### 開発者にとって重要なこと
複数モニターやウィンドウ管理を多用する人、PowerToys を作業効率化に使っているチームに関係します。キーボード割り当てやペースト動作の変更は、日々の操作感に直結します。VS Code や Teams など Electron 系アプリの操作にも影響する可能性があります。

### アクションプラン
- **⚠️** PowerToys を更新し、Grab And Move と Power Display を実際の作業環境で試す。
- Command Palette / Dock / Keyboard Manager の変更が既存ショートカットに影響しないか確認する。
- Electron / Chromium 系アプリを使っているなら、Advanced Paste の挙動を一度検証する。

### リファレンス
- [PowerToys 0.99 is here: new monitor controls, easier window management, and Dock upgrades](https://devblogs.microsoft.com/commandline/powertoys-0-99-is-here-new-monitor-controls-easier-window-management-and-dock-upgrades)
