## Introducing SQL MCP Server

- **Blog:** Azure SQL
- **URL:** https://devblogs.microsoft.com/azure-sql/introducing-sql-mcp-server

### 要約
SQL MCP Server を紹介する投稿です。企業がエージェントにデータアクセスを安全かつ機能豊富に提供する方法として位置づけられています。スキーマを直接公開せず、整合性を損なわず、脆弱な自然言語パースに頼らない点が強調されています。SQL データソースとエージェント連携の新しい接続方式を示しています。Azure SQL を使うチームや MCP ベースの統合に関心がある開発者向けです。

### 開発者にとって重要なこと
データベースをエージェントに公開したいが、権限・整合性・誤解釈を避けたいチームに関係します。SQL へのアクセスを直接渡すよりも、制御された層を作る設計判断に影響します。エージェント主導の検索や操作を検討している場合、アーキテクチャの候補になります。

### アクションプラン
- **⚠️** エージェントからデータベースへ直接アクセスしている箇所があれば、SQL MCP Server への置き換え可否を検討する。
- 権限、監査、整合性の要件を整理し、既存の自然言語ベース実装と比較する。
- 本文で示される導入条件や制約を確認して、PoC の対象にするか判断する。
