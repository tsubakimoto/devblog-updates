## MCP as Easy as 1-2-3: Introducing the Fluent API for MCP Apps

- **Blog:** Azure Sdk
- **URL:** https://devblogs.microsoft.com/azure-sdk/mcp-as-easy-as-1-2-3-introducing-the-fluent-api-for-mcp-apps

### 要約
Azure Functions の MCP extension で導入された MCP Apps を、より簡単に構築できるようにする投稿です。今回の更新では、.NET isolated worker 向けの fluent configuration API が追加されます。これにより、既存の MCP tool を MCP App に昇格させやすくなります。MCP Apps は、テキストだけでなく UI や静的資産を提供できる点が特徴です。AI agents との統合も前提にした構成です。

### 開発者にとって重要なこと
Azure Functions で MCP を使っているチームや、AI agent にリッチな UI を持たせたい開発者に関係します。個別の tool 実装から app レベルの体験に広げやすくなるため、構成の見直し余地があります。既存の .NET isolated worker ベースの実装を持つ場合は特に確認が必要です。

### アクションプラン
- Azure Functions の MCP extension を使っているなら、MCP tool を MCP App に昇格できるか確認する。
- .NET isolated worker の既存実装に fluent API を組み込めるか評価する。
- UI や静的資産を伴う agent 体験が必要かをチームで判断する。

- [MCP as Easy as 1-2-3: Introducing the Fluent API for MCP Apps](https://devblogs.microsoft.com/azure-sdk/mcp-as-easy-as-1-2-3-introducing-the-fluent-api-for-mcp-apps)

## MCP Apps now available in Copilot chat

- **Blog:** Microsoft365dev
- **URL:** https://devblogs.microsoft.com/microsoft365dev/mcp-apps-now-available-in-copilot-chat

### 要約
Microsoft 365 Copilot chat で MCP Apps が利用可能になったことを知らせる投稿です。MCP Apps か OpenAI Apps SDK を使って、チャット内にリッチなアプリ体験を組み込めます。単なる会話から、実際に操作できる UI 付きの体験へ広げる流れが示されています。Copilot と自然に連携する点が強調されています。Copilot chat を使うアプリ開発者向けの更新です。

### 開発者にとって重要なこと
Microsoft 365 の中で agent ベースの UI を提供したいチームに関係します。会話画面に埋め込む体験が増えるため、認証、状態管理、操作の透明性が重要になります。既存の Copilot 統合を持つ場合は、MCP Apps への対応可否を確認する価値があります。

### アクションプラン
- Copilot chat 内での操作が必要なユースケースを洗い出す。
- MCP Apps か OpenAI Apps SDK のどちらが適するかを比較する。
- **⚠️** ユーザー操作とデータアクセスの権限、監査要件を先に確認する。

### リファレンス
- [MCP Apps now available in Copilot chat](https://devblogs.microsoft.com/microsoft365dev/mcp-apps-now-available-in-copilot-chat)

## Your Migration's Source of Truth: The Modernization Assessment

- **Blog:** .NET
- **URL:** https://devblogs.microsoft.com/dotnet/your-migrations-source-of-truth-the-modernization-assessment

### 要約
GitHub Copilot の .NET と Java 向け modernization capabilities の中でも、assessment document に焦点を当てた投稿です。ツールはコードベース全体を解析し、assessment document を作成し、migration plan を組み立てます。さらに Azure infrastructure の provision まで支援できると説明されています。移行作業の出発点を自動化し、判断材料を整える流れです。大規模な移行を検討するチーム向けの内容です。

### 開発者にとって重要なこと
既存アプリの modernization を進めるチームに直接関係します。人手での棚卸しではなく、評価ドキュメントを移行計画の基準にできる点が重要です。移行対象の優先順位付けや、次の作業をそろえる用途で役立ちます。

### アクションプラン
- .NET または Java の移行案件で、assessment document を計画の起点にできるか確認する。
- 既存の移行手順と Copilot の出力を比較し、どこまで自動化できるか評価する。
- Azure infrastructure の provision を含める場合は、運用手順と権限を先に整理する。

### リファレンス
- [Your Migration's Source of Truth: The Modernization Assessment](https://devblogs.microsoft.com/dotnet/your-migrations-source-of-truth-the-modernization-assessment)

## ASP.NET Core 2.3 end of support announcement

- **Blog:** .NET
- **URL:** https://devblogs.microsoft.com/dotnet/aspnet-core-2-3-end-of-support

### 要約
ASP.NET Core 2.3 の end of support が 2027-04-07 になると告知する投稿です。Microsoft は 12 か月前の通知要件に従って、今回の案内を出しています。以降はセキュリティ修正、バグ修正、技術サポートが提供されません。現行のサポートポリシーに基づく正式な通知です。ASP.NET Core 2.3 を使っている開発者や運用チームに直接関係します。

### 開発者にとって重要なこと
古い ASP.NET Core 2.3 に依存しているアプリは、移行計画が必要になります。サポート終了後は脆弱性対応や不具合修正を受けられないため、リスクが上がります。長期運用中のシステムでは、互換性確認と移行スケジュールの確保が重要です。

### アクションプラン
- **⚠️** ASP.NET Core 2.3 を使っているアプリを洗い出し、移行対象を特定する。
- 2027-04-07 までの移行計画を作成し、依存サービスも含めて影響を確認する。
- サポート切れ後に残る構成がないか、運用・セキュリティ両面で再確認する。

### リファレンス
- [ASP.NET Core 2.3 end of support announcement](https://devblogs.microsoft.com/dotnet/aspnet-core-2-3-end-of-support)
